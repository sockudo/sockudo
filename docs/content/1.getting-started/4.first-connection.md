## 1) Connect a Client

::code-group

```js [V2 (Sockudo)]
import { SockudoClient } from "@sockudo/client";

const client = new SockudoClient("app-key", {
  wsHost: "127.0.0.1",
  wsPort: 6001,
  forceTLS: false,
});

const channel = client.subscribe("my-channel");
channel.bind("my-event", (payload) => {
  console.log("received:", payload);
});
```

```js [V1 (Pusher)]
import Pusher from "pusher-js";

const pusher = new Pusher("app-key", {
  cluster: "",
  wsHost: "127.0.0.1",
  wsPort: 6001,
  forceTLS: false,
  enabledTransports: ["ws"],
});

const channel = pusher.subscribe("my-channel");
channel.bind("my-event", (payload) => {
  console.log("received:", payload);
});
```

::

## 2) Trigger an Event (Server SDK)

::code-group

```js [Sockudo]
const { Sockudo } = require("sockudo");

const sockudo = new Sockudo({
  appId: "app-id",
  key: "app-key",
  secret: "app-secret",
  host: "127.0.0.1",
  port: 6001,
  useTLS: false,
});

await sockudo.trigger("my-channel", "my-event", {
  message: "hello from Sockudo",
  sentAt: Date.now(),
});
```

```js [Pusher]
const Pusher = require("pusher");

const pusher = new Pusher({
  appId: "app-id",
  key: "app-key",
  secret: "app-secret",
  host: "127.0.0.1",
  port: "6001",
  useTLS: false,
  cluster: "mt1",
});

await pusher.trigger("my-channel", "my-event", {
  message: "hello from Sockudo",
});
```

::

## 3) Verify Realtime Flow

When the backend trigger runs, your subscribed client should receive `my-event` immediately.

V2 clients will also see `serial` and `message_id` fields on each message (when connection recovery is enabled on the server).
