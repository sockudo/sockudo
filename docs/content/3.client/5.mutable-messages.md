## Overview

When a server-side caller updates, deletes, or appends to a versioned message, V2 clients subscribed to the channel receive a mutation event in real time. Official Sockudo client SDKs ship reducer helpers that turn a stream of these events into a single `MutableMessageState` value.

This is a **Protocol V2 only** feature. Set `protocolVersion: 2` to receive mutation events. V1 (Pusher-compatible) clients never receive mutable-message envelopes.

## Mutation Events

| Event name | Triggered when |
|---|---|
| `sockudo:message.create` | The initial publish for a versioned message is stored |
| `sockudo:message.update` | A server-side update is applied |
| `sockudo:message.delete` | A server-side soft delete is applied |
| `sockudo:message.append` | A server-side append is applied |

Every mutation event carries version metadata in `extras.headers`:

```json
{
  "event": "my-event",
  "channel": "docs:readme",
  "data": "Hello, world",
  "extras": {
    "headers": {
      "sockudo_action": "message.update",
      "sockudo_message_serial": "msg-123",
      "sockudo_version_serial": "v2",
      "sockudo_history_serial": 12,
      "sockudo_version_timestamp_ms": 1775292000123
    }
  }
}
```

## MutableMessageState Shape

The reducer helpers produce and consume `MutableMessageState` values. The shape is identical across all official SDKs:

| Field | Type | Description |
|---|---|---|
| `messageSerial` | `string` | Stable identity of the logical message |
| `action` | `MutableMessageAction` | Last applied action |
| `data` | `T \| null` | Current message data after applying the action |
| `event` | `string` | Event name as published |
| `serial` | `number?` | Channel serial from the broadcast envelope |
| `streamId` | `string?` | Durable history stream ID |
| `messageId` | `string?` | UUIDv4 message ID for deduplication |
| `versionSerial` | `string?` | Version serial of this mutation |
| `historySerial` | `number?` | Channel history serial for this version event |
| `versionTimestampMs` | `number?` | Unix millisecond timestamp of this version |

`MutableMessageAction` values: `"message.create"`, `"message.update"`, `"message.delete"`, `"message.append"`.

## Checking for Mutation Events

Use `isMutableMessageEvent()` before calling reducers to avoid errors on regular events.

::code-group

```ts [JavaScript]
import {
  isMutableMessageEvent,
  getMutableMessageInfo,
} from "@sockudo/client/versioned-messages";

channel.bind("my-event", (data, event) => {
  if (isMutableMessageEvent(event)) {
    const info = getMutableMessageInfo(event);
    console.log(info.action, info.messageSerial);
  }
});
```

```dart [Flutter]

```

## Reducing a Single Event

`reduceMutableMessageEvent(current, event)` takes the current state (or `null` for the first event) and returns the next state.

::code-group

```ts [JavaScript]
import { reduceMutableMessageEvent } from "@sockudo/client/versioned-messages";
import type { MutableMessageState } from "@sockudo/client/versioned-messages";

let state: MutableMessageState | null = null;

channel.bind("my-event", (data, event) => {
  if (isMutableMessageEvent(event)) {
    state = reduceMutableMessageEvent(state, event);
    console.log(state.action, state.data);
  }
});
```

```dart [Flutter]
import 'package:sockudo_flutter/src/versioned_messages.dart';

MutableMessageState? state;

channel.bind('my-event', (data, event) {
  if (isMutableMessageEvent(event)) {
    state = reduceMutableMessageEvent(state, event);
    print('${state!.action} ${state!.data}');
  }
});
```

```kotlin [Kotlin]

var state: MutableMessageState? = null

channel.bind("my-event") { data, event ->

    }
}
```

```swift [Swift]

}
```

```csharp [C#]
    }
});
```

## Reducing a Batch of Events

`reduceMutableMessageEvents(events)` applies a sequence of mutation events in order and returns the final state. Useful for seeding local state from a history page.

::code-group

```ts [JavaScript]
import { reduceMutableMessageEvents } from "@sockudo/client/versioned-messages";

// Seed state from a history page
const state = reduceMutableMessageEvents(historyItems);
if (state) {
  console.log(state.data);
}
```

```dart [Flutter]

final state = reduceMutableMessageEvents(historyItems);
```

## Append Semantics

For `message.append`, the reducer concatenates the fragment onto the current string base:

```
current.data = "Hello"
event.data   = ", world"
next.data    = "Hello, world"
```

The reducer throws if `current.data` is not a string when it encounters a `message.append` event. If you start receiving append events before you have an established string base, fetch the current state first using the proxy-backed pattern below and seed the reducer from there.

```
message.append requires an existing string base;
seed state from a create/update payload or latest-view history first
```

## Proxy-Backed Pattern

Client SDKs must not sign server REST API requests directly. Use a backend endpoint to proxy message retrieval:

```
POST /your-api/message-proxy
{ "channel": "docs:readme", "messageSerial": "msg-123", "action": "get_message" }
→ backend calls GET /apps/{appId}/channels/{channelName}/messages/{messageSerial}
→ returns { item: { ... } }
```

Example backend endpoint (Node.js):

```ts [Node.js backend]
import Sockudo from "@sockudo/http";

const client = new Sockudo({
  appId: process.env.APP_ID!,
  key: process.env.APP_KEY!,
  secret: process.env.APP_SECRET!,
  host: process.env.SOCKUDO_HOST!,
  port: Number(process.env.SOCKUDO_PORT),
  useTLS: false,
});

app.post("/message-proxy", async (req, res) => {
  const { channel, messageSerial, action } = req.body;

  if (action === "get_message") {
    const result = await client.getMessage(channel, messageSerial);
    return res.json(result);
  }

  if (action === "get_message_versions") {
    const result = await client.getMessageVersions(channel, messageSerial, req.body.params ?? {});
    return res.json(result);
  }

  res.status(400).json({ error: "unknown action" });
});
```

Frontend (JavaScript):

```ts [JavaScript]
import { reduceMutableMessageEvent } from "@sockudo/client/versioned-messages";
import type { MutableMessageState } from "@sockudo/client/versioned-messages";

async function fetchAndSeed(channel: string, messageSerial: string): Promise<MutableMessageState | null> {
  const res = await fetch("/message-proxy", {
    method: "POST",
    headers: { "content-type": "application/json" },
    body: JSON.stringify({ channel, messageSerial, action: "get_message" }),
  });
  const { item } = await res.json();
  if (!item) return null;
  // Seed a synthetic create event to bootstrap the state
  return {
    messageSerial: item.message_serial,
    action: item.action,
    data: item.data,
    event: item.event_name,
  };
}
```

## Full Example: Live-Updating Document

::code-group

```ts [JavaScript]
import Sockudo from "@sockudo/client";
import {
  isMutableMessageEvent,
  reduceMutableMessageEvent,
} from "@sockudo/client/versioned-messages";
import type { MutableMessageState } from "@sockudo/client/versioned-messages";

const client = new Sockudo("app-key", {
  wsHost: "127.0.0.1",
  wsPort: 6001,
  forceTLS: false,
  protocolVersion: 2,
});

const channel = client.subscribe("docs:readme");

let docState: MutableMessageState | null = null;

channel.bind("my-event", (data: unknown, event: unknown) => {
  if (!isMutableMessageEvent(event as any)) return;

  try {
    docState = reduceMutableMessageEvent(docState, event as any);
    renderDoc(docState);
  } catch (err) {
    // message.append before a string base: fetch current state and retry
    if (docState === null && (event as any).extras?.headers?.sockudo_action === "message.append") {
      fetchAndSeed("docs:readme", (event as any).extras.headers.sockudo_message_serial)
        .then((seeded) => {
          docState = seeded;
          docState = reduceMutableMessageEvent(docState, event as any);
          renderDoc(docState);
        });
    }
  }
});

function renderDoc(state: MutableMessageState | null) {
  if (!state) return;
  if (state.action === "message.delete") {
    console.log("[deleted]");
  } else {
    console.log(state.data);
  }
}
```

## Seeding State from History

When a client joins a channel mid-stream and needs to display the current state of a mutable message, fetch the latest visible version through your backend proxy, then seed the reducer before applying new live events.

::code-group

```ts [JavaScript]
import { reduceMutableMessageEvents } from "@sockudo/client/versioned-messages";
import type { MutableMessageState, SockudoEvent } from "@sockudo/client";

async function seedFromHistory(channel: string, messageSerial: string) {
  const res = await fetch("/message-proxy", {
    method: "POST",
    headers: { "content-type": "application/json" },
    body: JSON.stringify({
      channel,
      messageSerial,
      action: "get_message_versions",
      params: { direction: "oldest_first", limit: 100 },
    }),
  });
  const { items }: { items: SockudoEvent[] } = await res.json();
  return reduceMutableMessageEvents(items);
}
```

```dart [Flutter]
import 'package:sockudo_flutter/src/versioned_messages.dart';
import 'package:http/http.dart' as http;
import 'dart:convert';

Future<MutableMessageState?> seedFromHistory(
  String channel,
  String messageSerial,
) async {
  final response = await http.post(
    Uri.parse('/message-proxy'),
    headers: {'content-type': 'application/json'},
    body: jsonEncode({
      'channel': channel,
      'messageSerial': messageSerial,
      'action': 'get_message_versions',
      'params': {'direction': 'oldest_first', 'limit': 100},
    }),
  );
  final body = jsonDecode(response.body) as Map<String, dynamic>;
  final items = (body['items'] as List).cast<Map<String, dynamic>>();
  // Map to SockudoEvent instances and reduce
  return reduceMutableMessageEvents(items.map(SockudoEvent.fromMap).toList());
}
```

## Error Handling Notes

- `reduceMutableMessageEvent` throws if the event is not a valid mutable message event.
- `reduceMutableMessageEvent` throws if `current.messageSerial` does not match the incoming event's `messageSerial`.
- `reduceMutableMessageEvent` throws on `message.append` when `current.data` is not a string.

In Swift, `reduceMutableMessageEvent` is a throwing function. In Kotlin and Dart, it throws `StateError`. In JavaScript and C#, it throws `Error` / `InvalidOperationException`.

## V1 Compatibility

V1 clients (Pusher-compatible, `?protocol=1`) never receive `sockudo:message.update`, `sockudo:message.delete`, or `sockudo:message.append` envelopes. The mutable-message reducer helpers are not needed and have no effect for V1 connections.
