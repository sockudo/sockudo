## Official Sockudo Server SDKs

Sockudo provides official server SDKs for 8 languages. All support `idempotency_key` for safe publish retries.

| Language | Package | Install |
|---|---|---|
| Node.js | `sockudo` | `npm install sockudo` |
| PHP | `sockudo/sockudo-php-server` | `composer require sockudo/sockudo-php-server` |
| Ruby | `sockudo` | `gem install sockudo` |
| Go | `github.com/sockudo/sockudo-http-go/v5` | `go get github.com/sockudo/sockudo-http-go/v5` |
| Rust | `sockudo-http` | `cargo add sockudo-http` |
| Java | `io.sockudo:sockudo-http-java` | Gradle/Maven |
| .NET | `SockudoServer` | NuGet |
| Swift | `Sockudo` | Swift Package Manager |

## Usage Examples

::code-group

```js [Node.js]
const { Sockudo } = require("sockudo");

const sockudo = new Sockudo({
  appId: "app-id",
  key: "app-key",
  secret: "app-secret",
  host: "127.0.0.1",
  port: 6001,
  useTLS: false,
});

// Publish an event
await sockudo.trigger("orders", "order.updated", { id: 42 });

// Publish with idempotency key (safe retries)
await sockudo.trigger("orders", "order.updated", { id: 42 }, {
  idempotency_key: "unique-request-id-123"
});

// Batch publish
await sockudo.triggerBatch([
  { channel: "orders", name: "order.created", data: { id: 1 } },
  { channel: "orders", name: "order.created", data: { id: 2 } },
]);
```

```php [PHP]
use Sockudo\Sockudo;

$sockudo = new Sockudo([
    'app_id' => 'app-id',
    'key' => 'app-key',
    'secret' => 'app-secret',
    'host' => '127.0.0.1',
    'port' => 6001,
    'useTLS' => false,
]);

$sockudo->trigger('orders', 'order.updated', ['id' => 42]);

// With idempotency key
$sockudo->trigger('orders', 'order.updated', ['id' => 42], [], null, [
    'idempotency_key' => 'unique-request-id-123'
]);
```

```go [Go]
package main

import sockudo "github.com/sockudo/sockudo-http-go/v5"

func main() {
    client := sockudo.ClientConfig{
        AppID:   "app-id",
        Key:     "app-key",
        Secret:  "app-secret",
        Host:    "127.0.0.1",
        Port:    "6001",
        Secure:  false,
    }
    s, _ := sockudo.New(client)

    s.Trigger("orders", "order.updated", map[string]interface{}{
        "id": 42,
    })
}
```

```rust [Rust]
use sockudo_http::Sockudo;

let sockudo = Sockudo::new("app-id", "app-key", "app-secret")
    .host("127.0.0.1")
    .port(6001);

sockudo.trigger("orders", "order.updated", r#"{"id": 42}"#).await?;
```

## Auth Endpoint Helpers

All server SDKs provide auth helpers for private/presence channels:

```js
// Node.js
app.post("/sockudo/auth", (req, res) => {
  const { socket_id, channel_name } = req.body;

  if (channel_name.startsWith("presence-")) {
    const presenceData = {
      user_id: "user-123",
      user_info: { name: "Alice" },
    };
    return res.send(sockudo.authorizeChannel(socket_id, channel_name, presenceData));
  }

  return res.send(sockudo.authorizeChannel(socket_id, channel_name));
});
```

The auth signature format is identical to Pusher's HMAC-SHA256 scheme. Auth works the same regardless of protocol version (V1 or V2).

## Webhooks

Verify webhook signatures with the `X-Sockudo-Signature` header against the raw body and app secret:

```js
const isValid = sockudo.webhook(req.headers, req.rawBody);
```

## Idempotency Key

All Sockudo server SDKs support `idempotency_key` for safe publish retries. When provided, the server ensures the event is broadcast only once even if the HTTP request is retried:

- Pass `idempotency_key: "unique-id"` as a string for an explicit key
- Pass `idempotency_key: true` to auto-generate a UUIDv4

The server caches keys and returns the same response for duplicates.

## Pusher Server SDK Compatibility

If you already use Pusher server SDKs, they work with Sockudo's HTTP API by changing the host/port:

```js
// Using the official Pusher Node.js SDK
const Pusher = require("pusher");

const pusher = new Pusher({
  appId: "app-id",
  key: "app-key",
  secret: "app-secret",
  host: "127.0.0.1",
  port: "6001",
  useTLS: false,
  cluster: "mt1",
});

await pusher.trigger("orders", "order.updated", { id: 42 });
```

Pusher server SDKs do not support `idempotency_key`. For that feature, use the Sockudo server SDKs.

## SDK-Agnostic Fallback

Call the Sockudo HTTP API directly with signed query params:

- `POST /apps/{appId}/events`
- `POST /apps/{appId}/batch_events`

See [HTTP API](/server/http-api) for details.
